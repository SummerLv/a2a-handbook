# A2A Handbook 文档风格指南

> 统一视觉风格，提升阅读体验

## 📝 标题规范

```markdown
# 一级标题 - 文档主题

> 文档摘要

## 二级标题 - 核心章节（可配 emoji）
### 三级标题 - 子章节
```

**原则**：清晰描述性，避免跳级，emoji 仅用于章节标题。

---

## 🎨 视觉元素

### Emoji 使用

```
📚 文档：📖 概念 | 💡 示例 | 🎯 案例 | 📝 指南
💻 代码：🔧 工具 | 🔒 安全 | 🧪 测试
🚀 状态：✅ 完成 | ⚠️ 警告 | 🎉 成功
```

**原则**：首页章节可用，正文节制使用，同类内容保持一致。

### 代码块规范

```bash
# 必须指定语言
curl -s https://api.example.com

# Python - 添加关键注释
class A2AClient:
    def __init__(self, base_url):
        self.base_url = base_url
```

**要求**：指定语言，关键代码注释，示例数据有意义，>30行拆分。

### 表格规范

```markdown
| 概念 | 说明 | 示例 |
|------|------|------|
| **Agent Card** | 能力声明 | `/.well-known/agent.json` |
```

**原则**：表头粗体，≤3列优先，复杂结构用列表。

## ✍️ 写作风格

### 语言规范
- **中文为主**，术语保留英文（Agent Card、JSON-RPC）
- **代码注释**英文，说明文字中文

### 语气要求
- **指导性**：主动语态，"发送请求"、"配置参数"
- **简洁**：避免冗余词汇
- **友好**：关键节点添加鼓励

### 格式化
- **强调**：粗体 `**重点**` 优先
- **代码**：行内用 `` `code` ``
- **路径**：代码格式 `/path/to/file`
