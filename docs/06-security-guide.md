## 安全威胁分类

### 1. 注入攻击类

| 威胁类型 | 风险等级 | 描述 |
|---------|---------|------|
| **Prompt Injection** | 🔴 高 | 通过构造输入绕过系统限制 |
| **Context Poisoning** | 🔴 高 | 修改 Agent Card 注入恶意上下文 |
| **代码注入** | 🟠 中 | 通过文件或数据字段注入可执行代码 |

### 2. 身份冒充类

| 威胁类型 | 风险等级 | 描述 |
|---------|---------|------|
| **Agent Impersonation** | 🔴 高 | 攻击者冒充合法 Agent |
| **Agent Card Shadowing** | 🟠 中 | 创建与合法 Agent 相似的恶意 Agent |
| **凭证泄露** | 🔴 高 | Token、API Key 等被窃取 |

### 3. 数据泄露类

| 威胁类型 | 风险等级 | 描述 |
|---------|---------|------|
| **敏感信息泄露** | 🟠 中 | Agent Card 暴露内部信息 |
| **日志泄露** | 🟡 低 | 日志中包含敏感数据 |
| **响应泄露** | 🟠 中 | 响应中包含系统内部信息 |

### 4. 拒绝服务类

| 威胁类型 | 风险等级 | 描述 |
|---------|---------|------|
| **请求洪泛** | 🟠 中 | 大量请求耗尽资源 |
| **资源耗尽** | 🟠 中 | 大文件或复杂任务消耗资源 |

## 认证与授权

### 🔐 Bearer Token 认证

**核心原则**：Access Token 短期有效（30分钟），Refresh Token 长期有效（7天），支持刷新和撤销。

### 🎫 OAuth 2.0 集成

**支持流程**：Authorization Code（推荐 PKCE）、Client Credentials（服务间通信）、Refresh Token。

### 🔑 API Key 管理

**最佳实践**：格式 `prefix_keyId_keySecret`，存储 Hash 值，支持过期、撤销、轮换。

### 🛡️ Scope 权限控制

**最小权限原则**：默认拒绝所有操作，仅授予必要权限。

## 注入攻击防护

### 💉 Prompt Injection 防御

**检测策略**：指令覆盖、角色扮演、权限绕过、编码绕过检测。

**多层防御**：输入检测 → 输入清理 → 系统提示隔离 → 结构化输入。

### ✅ 输入验证

**验证规则**：文本最大 100KB，文件最大 10MB，数据嵌套深度 ≤10。

### 🏖️ 沙箱隔离

**隔离级别**：进程隔离（低风险）、容器隔离（高风险）、VM 隔离（不可信）。

---

## 身份验证

### 🤖 Agent 身份管理

**注册流程**：生成密钥对 → 提交公钥注册 → 获取 Agent ID。

**挑战-响应认证**：请求挑战 → 签名挑战 → 验证响应。

### 🔍 Shadowing 检测

**检测方法**：名称相似度、URL 相似度、Typosquatting 检查。相似度 > 90% 为高风险。

### 📜 证书验证

**证书链验证**：有效期 → 颁发者 → 签名 → 撤销状态。支持 mTLS 双向验证。

## 基础设施安全

### 🔒 TLS/HTTPS 配置

**最低要求**：TLS 1.2+，强加密套件，HSTS 启用，证书自动续期。

### ⏱️ 速率限制

**算法选择**：滑动窗口（精确控制）、令牌桶（允许突发）、漏桶（平滑流量）。

### 📝 日志和审计

**审计事件**：认证、消息、任务、安全事件。日志包含时间戳、事件类型、主体、结果、风险等级。

### 📊 监控和告警

**核心指标**：认证（失败登录、活跃会话）、请求（QPS、延迟、错误率）、安全（注入尝试、异常检测）、资源（CPU、内存）。

**告警级别**：🔴 Critical（立即响应）、🟠 Warning（1小时内）、🟡 Info（24小时内）。

## 零信任架构

### 🎯 最小权限原则

**默认拒绝**：未明确授权的操作一律拒绝，敏感操作需要额外验证。

### 🔄 持续验证

**验证频率**：每 5 分钟。验证项目：IP 一致性、Token 有效性、用户状态、权限变更。

### 🧱 微隔离

**隔离策略**：可信 Agent（进程隔离，内部网络）、不可信 Agent（容器隔离，沙箱网络）、外部 Agent（容器隔离，无网络）。

## 安全检查清单

### ✅ 部署前检查

**认证与授权**：所有端点需认证，Token 过期/刷新/撤销机制，Scope 权限最小化。

**Agent Card 安全**：Agent Card 已签名，敏感信息已过滤，版本控制已配置。

**注入防护**：输入验证已实现，Prompt Injection 检测已启用，文件上传有限制，代码执行有沙箱。

**基础设施**：HTTPS 已启用（TLS 1.2+），速率限制已启用，日志审计已配置，监控告警已配置。

**零信任**：最小权限原则已应用，持续验证已实现，微隔离已配置。

### 🔍 运行时监控

**监控项**：认证监控、请求监控、异常监控、资源监控。

### 🚨 事件响应

**响应优先级**：P0 紧急（<15分钟）、P1 高（<1小时）、P2 中（<4小时）、P3 低（<24小时）。

**响应步骤**：检测 → 遏制 → 根除 → 恢复 → 总结。
